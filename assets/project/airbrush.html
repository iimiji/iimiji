Airbrush

Drawing application system that allows user to use gestures and movement to drawing

Team 13 | IAT267

Concept

Airbrush is a digital painting application and tool that simulates a spraypaint can and teaches individuals how to do graffiti-style artwork by tracing and giving step-by-step tutorials. This will prove to be a non-destructive alternative to traditional graffiti-style artwork and caters towards artists between the age of 15-20 who have taken interest in this medium. The remote controller is built from recycled material and uses radio transmission to communicate with our computer interface. Our project sets itself apart from other digital spraypaint simulations by having a unique controller made with a real spray paint can to enhance the experience.

Prototype: In the first iteration of our project, we are planning to make a remote painting brush, however, after receiving feedback, we elevate our concept based on rethinking the values it can bring to people. We finalized a clear definition of the idea of creating a digital spray can to promote this form of art in a non-destructive way.

Reflection

There were a few things our team would have liked to change if we had more time. First, the accuracy of the system. Using an accelerometer to determine position and velocity is possible but causes a lot of issues. Based on our research, there are other methods of determining position that we would have liked to try. One in particular involved using two ultrasonic sensors along the axes of the physical screen to create a non-right triangle, so that you can calculate position by solving the triangle with math formulas. Also, the application would benefit from having buttons and a menu system so that users could select and navigate through the different screens inside of the application using the spray paint controller instead of pressing buttons on the keyboard. For demonstration purposes, the application would also be better on a bigger scale using a projector instead of a computer screen.
I was in charge of producing templates for the final application in this project. In order to provide the tutorial of graffiti by easy user experience, guidelines were created along circles and numbers to reduce the transparency of the artwork and to allow the user to follow and complete the graffiti artwork. Later, four backgrounds were created through Figma to allow users to select free drawing and drawing tutorials 1 and 2 through buttons in processing. In addition, we set up a user persona to more clearly communicate to which user groups our project can help through story setting. When I first started this project, I thought it would be safe to produce it in a similar way to an existing project, as many other teams did. This is because I thought it would be more difficult than I thought to set the direction of the project first and solve the idea with technology. However, even though it is a project that does not exist before, we have created a complete product that is as close to our idea as possible through a variety of attempts in a given material and in a given time. Collaborating with team members who are interested in the project and have smooth communication was a good experience for me to feel that there are many advantages of the team project and things to learn from the team project.
